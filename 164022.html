1. Business Requirements

1.1. Background

У сучасних умовах мобільності та цифровізації зростає попит на зручні онлайн-інструменти для планування подорожей. Бронювання авіаквитків онлайн стало стандартом серед користувачів, однак багато платформ залишаються складними у користуванні або не забезпечують повного циклу обслуговування. Наша компанія прагне створити інтуїтивно зрозумілий додаток, який дозволить користувачам швидко знайти, забронювати та оплатити авіаквитки, отримуючи актуальну інформацію в реальному часі.

1.2. Business Opportunity

  • Створення конкурентноспроможного рішення на ринку цифрових авіапослуг

  • Залучення партнерства з авіакомпаніями, агрегаторами рейсів, туристичними сервісами

  • Побудова довгострокових відносин з користувачами через програми лояльності

1.3. Business Objectives and Success Criteria

Мета проєкту:

  • Розробити інтуїтивно зрозумілий мобільний додаток для бронювання авіаквитків, який забезпечить простий пошук, порівння та оплату квитків в режимі реального часу

  • Інтегрувати додаток з провідними системами бронювання та API авіакомпаній, щоб забезпечити актуальні ціни.

Критерії успіху:

  • Підтримка покриває 90% запитів користувачів протягом перших 60 хвилин.

  • Досягнення ≥90% позитивних відгуків користувачів у перші 3 місяці після запуску.

  • Досягти щонайменше 50 000 активних користувачів у перший рік запуску

  • Інтеграція з принаймні трьома глобальними постачальниками даних про авіарейси.

1.4. Customer or Market Needs

Користувачі потребують:

  • Зручного інтерфейсу для швидкого пошуку рейсів.

  • Можливості фільтрувати за ціною, часом вильоту, авіакомпаніями.

  • Прозорої системи цін та відсутності прихованих комісій.

  • Підтримки української мови та великий вибір платіжних систем

  • Можливості збереження квитків у додатку та отримання push-сповіщень щодо змін у рейсах.

1.5. Business Risks

  • Висока конкуренція на ринку

  • Регуляторні обмеження щодо зберігання персональних даних

  • Інтеграційні ризики під час роботи з API зовнішніх провайдерів рейсів.

  • Залежність від курсів валют, що може впливати на фінансові розрахунки.

  • Кіберзагрози

2. Vision of the Solution

2.1. Vision Statement

Ми прагнемо створити інтуїтивно зрозумілий,  технологічно сучасний додаток для бронювання авіаквитків, який забезпечить користувачам швидкий доступ до актуальних пропозицій з усього світу. Рішення має стати надійним супутником у плануванні подорожей, дозволяючи легко знаходити рейси, здійснювати оплату, зберігати квитки, отримувати сповіщення . Основна ціль — надати користувачам максимально зручний інструмент для подорожей, який відповідає їхнім реальним потребам та очікуванням.

2.2. Major Features

  • Пошук авіарейсів за напрямком, датою, кількістю пасажирів, класом обслуговування тощо.

  • Фільтри: ціна, авіакомпанія, кількість пересадок, час вильоту/прильоту.

  • Бронювання квитків у реальному часі через інтеграцію з глобальними системами (GDS).

  • Оплата онлайн (банківські картки, Apple Pay, Google Pay).

  • Підтримка кількох мов, включаючи українську та англійську.

  • Збереження електронного квитка в додатку з можливістю офлайн-доступу.

  • Push-сповіщення про зміни в розкладі, початок реєстрації, нагадування про рейс.

  • Особистий кабінет з історією бронювань, профілем пасажира, налаштуваннями.

  • Інтеграція з бонусними програмами авіакомпаній .

  • Підтримка через чат-бота

2.3. Assumptions and Dependencies

Припущення:

  • Користувачі мають доступ до інтернету та володіють базовими цифровими навичками.

  • Постачальники авіарейсів надають відкритий або комерційний доступ до своїх API.

  • Платіжні системи дозволяють інтеграцію через публічні SDK/API.

  • Всі авіакомпанії-партнери мають актуальні розклади та політики, які можна оновлювати в режимі реального часу.

Залежності:

  • Надійна інтеграція з API постачальників даних про авіарейси

  • Інтеграція з банківськими платіжними шлюзами для здійснення транзакцій.

  • Дотримання правових норм, включаючи захист персональних даних

  • Постійне оновлення мобільних ОС (Android, iOS) для підтримки стабільної роботи додатку.

3. Scope and Limitations

3.1. Scope of Initial Release

Початковий реліз включатиме базовий функціонал, необхідний для повноцінного бронювання авіаквитків:

  • Мобільний додаток (Android та iOS).

  • Пошук авіарейсів за ключовими параметрами: маршрут, дати, кількість пасажирів.

  • Фільтрація результатів (ціна, час, авіакомпанія, пересадки).

  • Інтеграція з API постачальника рейсів .

  • Можливість бронювання та купівлі квитків.

  • Онлайн-оплата (банківські картки, Apple Pay, Google Pay).

  • Особистий кабінет користувача з історією бронювань.

  • Збереження електронного квитка та відображення QR/штрих-коду.

  • Push-сповіщення про статус рейсів.

  • Українська та англійська мови інтерфейсу.

  • Базова підтримка користувачів через чат

3.2. Scope of Subsequent Releases

Наступні релізи передбачають розширення функціоналу:

  • Пошук комбінованих маршрутів (з декількома перевізниками або складними пересадками).

  • Віртуальний помічник для планування подорожей.

  • Програма лояльності/бонусна система для зареєстрованих користувачів.

  • Інтеграція з авіакомпаніями для реєстрації на рейс через додаток.

3.3. Limitations and Exclusions

  • Наразі не передбачається підтримка бронювання групових або корпоративних поїздок.

  • Відсутність офлайн-функціоналу для повного циклу бронювання (інтернет з’єднання необхідне).

  • Всі дії з персональними даними будуть здійснюватись згідно з чинним законодавством, однак обмеження деяких регіонів (наприклад, санкційні обмеження) можуть впливати на доступ до сервісу.

4.2. Project Priorities

  1. Інтерфейс має бути максимально зручним та інтуїтивно зрозумілим.

  2. Надійна інтеграція з API постачальників рейсів та платіжними сервісами.

  3. Безпека персональних і фінансових даних

  4. Масштабованість системи — врахування можливості подальшого розширення функціоналу

  5. Підтримка української мови та можливість вибору методів оплати.

4.3. Operating Environment

  • Клієнтські платформи: Android/iOS

  • Серверне середовище: хмарна інфраструктура з можливістю масштабування.

  • Інтеграційне середовище: REST API

  • Безпека: зберігання даних відповідно до GDPR.

  • Мова інтерфейсу: українська, англійська (початково).

  • Доступність: додаток доступний 24/7
